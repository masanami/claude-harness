## 要件モード（機能仕様 → 親チケット）

### Step 1: 機能仕様ドキュメントの読み込み

引数のパスから機能仕様ドキュメントを読み込む。ドキュメントには以下が含まれる前提:

- 概要・背景・ユーザーストーリー
- 機能要件 / 非機能要件
- **クリティカル設計決定**（該当する場合）
- 機能全体の設計（複雑な機能のみ）
- 受入基準

足りないセクションがあれば `/define-feature` への差し戻しを案内する。

### Step 2: Issue 本文の構成

Issue 本文は次の形にする（機能仕様の内容をそのまま使う）:

```text
> 機能仕様: {入力された機能仕様ドキュメントパス}

{機能仕様ドキュメントの本文をそのまま貼り付け}
```

機能仕様へのリンクを冒頭に置き、本文を再掲する（GitHub 上での閲覧性のため）。

### Step 3: GitHub Issue 作成

```bash
gh issue create \
  --title "{機能仕様のタイトル(機能名)}" \
  --body "$(cat <<'EOF'
{Step 2 で構成した本文}
EOF
)" \
  --label "requirement"
```

ラベルはプロジェクトの慣習に従う（`requirement` が無ければ作成 or 別名を使用）。

### Step 4: 完了報告（要件モード）

```text
## 要件チケット作成 完了

- 親要件チケット: #{番号}
- 機能仕様: {入力された機能仕様ドキュメントパス}

次のステップ:
# 実装チケットに分解
/create-ticket {番号}
```
