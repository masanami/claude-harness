# Self Review

現在のブランチの変更差分に対してセルフレビューを実施します。

## 手順

### 1. 変更差分の取得

```bash
# 現在のブランチの変更を確認
git log origin/main..HEAD --oneline
git diff origin/main..HEAD
```

### 2. コードレビューの実施

native の `/code-review` を実行し、正確性・品質・セキュリティ・簡潔化の観点をレビュー:

- バグ・正確性
- 品質・セキュリティ
- 簡潔化・再利用

> コーディング規約準拠やテスト充足などプロジェクト固有の観点を強めたい場合は、`/code-review` の指摘に加えて CLAUDE.md の規約を参照して補足する。

### 3. 設計レビューの実施

`design-reviewer` エージェントに委譲して設計レビューを実施:

- 依存方向の検証
- パッケージ境界の検証
- Feature-based 構造の検証
- プロジェクト固有の設計検証

### 4. レビュー結果の報告

以下の形式で報告:

1. **問題点**: 修正が必要な箇所
2. **改善提案**: より良い実装の提案（任意）
3. **良い点**: 評価できる実装（任意）

### 5. 問題がある場合

指摘事項を修正し、再度セルフレビューを実施。すべての問題が解消されるまで繰り返す。
